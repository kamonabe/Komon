### 閾値レベル列挙型
```python
class ThresholdLevel(Enum):
    NORMAL = "normal"      # 正常
    WARNING = "warning"    # 警告
    ALERT = "alert"        # 警戒
    CRITICAL = "critical"  # 緊急
```

## 正確性プロパティ

### [PROP-001] 閾値の順序性
**プロパティ**: すべてのメトリクスタイプについて、警告閾値 < 警戒閾値 < 緊急閾値  
**検証方法**: hypothesisによるプロパティベーステスト

### [PROP-002] レベル判定の正確性
**プロパティ**: 値Vと閾値(W, A, C)が与えられたとき:
- V < W の場合: レベル = NORMAL
- W ≤ V < A の場合: レベル = WARNING
- A ≤ V < C の場合: レベル = ALERT
- C ≤ V の場合: レベル = CRITICAL

**検証方法**: すべての範囲をカバーするプロパティベーステスト

### [PROP-003] 後方互換性
**プロパティ**: 従来の単一閾値Tは、3段階(T, T, T)と等価である  
**検証方法**: 出力を比較するユニットテスト

### [PROP-004] メッセージの一貫性
**プロパティ**: 各閾値レベルは、正確に1つの絵文字とメッセージテンプレートにマッピングされる  
**検証方法**: すべてのレベルに対するユニットテスト

### [PROP-005] 設定の検証
**プロパティ**: 無効な閾値設定（例: 警告 > 警戒）はValidationErrorを発生させる  
**検証方法**: 無効な設定を使用したユニットテスト

## テスト戦略

### プロパティベーステスト（hypothesis）
1. 閾値順序の検証
2. すべての値範囲に対するレベル判定
3. 設定の正規化（従来形式 → 3段階形式）

### ユニットテスト
1. 有効な3段階設定の読み込み
2. 従来設定の後方互換性
3. 無効な設定の拒否
4. メッセージテンプレートの選択
5. 絵文字の割り当て

### 統合テスト
1. 3段階レベルでのエンドツーエンド閾値検知
2. 適切なメッセージでの通知生成
3. 設定ファイルの読み込みと検証

## エッジケース

1. **閾値ちょうどの値**: レベル判定には >= を使用
2. **閾値レベルの欠落**: 従来の単一閾値にフォールバック
3. **無効な順序**: 起動時にValidationErrorを発生
4. **負の値**: NORMALレベルとして扱う
5. **100超の値**: 閾値ロジックを適用（CPUバーストシナリオ用）

## パフォーマンス考慮事項

- 閾値判定: O(1)の比較演算
- 設定検証: 起動時に1回のみ
- 追加のメモリオーバーヘッドなし（既存構造を再利用）

## セキュリティ考慮事項

- 閾値が数値で妥当な範囲（0-200）内であることを検証
- 適切なYAMLパースによる設定インジェクションの防止
- 監査証跡のための設定検証エラーのログ記録
