## テンプレート使用ガイド

### Front Matter（必須）
- `title`: 機能名と「実装タスク」を含める
- `feature`: requirements.md, design.mdと同じ値を使用
- `status`: todo / in-progress / completed / blocked
- `created`: 作成日（YYYY-MM-DD形式）
- `updated`: 最終更新日（YYYY-MM-DD形式）

### タスクチェックリスト
- タスクは実装順に並べる
- 各タスクに検証対象の要件（AC-XXX）を明記
- プロパティテストとユニットテストを別タスクとして分ける
- 必ず「チェックポイント」タスクを含める
- 最後に「ドキュメント更新」タスクを含める

### タスクの粒度
- 1つのタスクは1-2時間で完了できる粒度
- 大きすぎる場合はサブタスクに分割
- テストは実装タスクの直後に配置

### 要件とのトレーサビリティ
- 各タスクに `_要件: AC-XXX.X_` を必ず記載
- プロパティテストには検証対象のプロパティ番号を記載
- 要件定義との対応関係を明確に

### 品質チェックリスト
- [ ] Front Matterが全て記入されている
- [ ] タスクが実装順に並んでいる
- [ ] 各タスクに要件（AC-XXX）が明記されている
- [ ] プロパティテストタスクが含まれている
- [ ] ユニットテストタスクが含まれている
- [ ] チェックポイントタスクが含まれている
- [ ] ドキュメント更新タスクが最後にある
- [ ] 全てのタスクが `[ ]` または `[x]` で始まっている
