## 実装ワークフロー

```
1. future-ideas.md でアイデアを検討
   ↓
2. 実装を決定したら、このファイルにタスク化
   ↓
3. タスクステータスを更新しながら実装
   ↓
4. テスト・レビュー
   ↓
5. ステータスを 🟢 Done に更新
   ↓
6. CHANGELOG.md に記録
   ↓
7. future-ideas.md のステータスを「実装済み」に更新
```

## 優先順位の判断基準

**High Priority**:
- ユーザー体験を大きく改善する
- 実装コストが低〜中程度
- Komonの思想を強化する
- 既存機能との親和性が高い

**Medium Priority**:
- 便利だが必須ではない
- 実装コストが中程度
- 特定のユースケースで有用

**Low Priority**:
- 実装コストが高い
- 効果が不確実
- まずは他の機能を優先すべき
