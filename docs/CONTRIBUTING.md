# Komonへの貢献

Komonへの貢献に興味を持っていただき、ありがとうございます！

## 🐧 プラットフォーム方針

Komonは**Linux環境専用**として開発されています。

### 作者の方針

- Linux環境での実用性と品質を最優先
- Windows/macOS版は作者自身では開発しない
- 他のプラットフォームへの移植は、フォークによる独立プロジェクトとして歓迎

### なぜLinux専用なのか？

1. **実用性**: 作者が実際にLinuxサーバで使用するために開発
2. **品質**: 限られたリソースを一つのプラットフォームに集中
3. **シンプルさ**: クロスプラットフォーム対応の複雑さを避ける
4. **Linux機能の活用**: systemd、dnf/apt等を積極的に利用

## 🤝 貢献の種類

### 歓迎する貢献

1. **Linux環境での改善**
   - バグ修正
   - 新しいLinuxディストリビューション対応
   - パフォーマンス改善
   - ドキュメント改善

2. **機能追加**
   - Linux環境での新しい監視項目
   - 通知手段の追加
   - ログ分析の改善

3. **テストの追加**
   - ユニットテスト
   - 統合テスト
   - CI/CDの改善

### 他のプラットフォームへの移植について

Windows版やmacOS版を作りたい場合：

1. **フォークして独立プロジェクトとして開発してください**
2. 完成したら、ぜひお知らせください
3. 本家READMEにリンクを掲載させていただきます

例：
- `Komon-Windows` - Windows版フォーク
- `Komon-macOS` - macOS版フォーク

## 📝 貢献の流れ

### 1. Issue を作成

まず、何を改善したいか、何を追加したいかをIssueで議論しましょう。

### 2. Fork & Clone

```bash
# リポジトリをフォーク後
git clone https://github.com/your-username/Komon.git
cd Komon
```

### 3. ブランチを作成

```bash
git checkout -b feature/your-feature-name
```

### 4. 開発

```bash
# 開発モードでインストール
pip install -e .

# 変更を加える
# テストを追加
```

### 5. テスト

```bash
# テストを実行（今後実装予定）
pytest tests/

# 実際に動作確認
komon status
python scripts/main.py
```

### 6. コミット

```bash
git add .
git commit -m "feat: 新機能の説明"
```

コミットメッセージは以下の形式を推奨：
- `feat:` - 新機能
- `fix:` - バグ修正
- `docs:` - ドキュメント
- `refactor:` - リファクタリング
- `test:` - テスト追加

### 7. Pull Request

GitHubでPull Requestを作成してください。

## 🎯 開発ガイドライン

### コーディングスタイル

- PEP 8に準拠
- 日本語コメント・メッセージOK（Komonの特徴です）
- 絵文字の使用を推奨（ユーザーフレンドリーに）

### 設計原則

1. **シンプルさ**: 複雑な実装より、シンプルで理解しやすいコードを優先
2. **実用性**: 実際に使える機能を重視
3. **拡張性**: 新しい機能を追加しやすい設計
4. **Linux特化**: クロスプラットフォーム対応は考慮しない

### ドキュメント

コードを変更したら、以下も更新してください：

- `docs/README.md` - 機能説明
- `.kiro/specs/komon-system.md` - 仕様書
- `docs/CHANGELOG.md` - 変更履歴

## 🔍 仕様駆動開発

Komonは仕様駆動開発を採用しています。

1. **仕様を書く**: `.kiro/specs/komon-system.md`に追記
2. **実装する**: `src/komon/`にコードを追加
3. **テストを書く**: `tests/`にテストを追加
4. **ドキュメント更新**: 関連ドキュメントを更新
